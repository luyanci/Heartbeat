# Heartbeat

如果我 `5` 天没有对此仓库提交过更改，说明作为人类实体的我可能已经死亡。

## 简介

手动更新的程序员“心跳包”，用于确认该 Github 账号背后的人类实体是否存活。通过对 fork 仓库提交更改的方式，既确保可以每日提交一次进行活跃，也可以确保不会对自己的 Contributions 墙产生污染。

虽然我想可能已经有人做过类似的仓库了，不过还是我自己再建一个仓库比较稳妥。

## 使用方法

1. fork 本仓库
3. 规定一个心跳包超时时间
2. 使用 Github Mobile 客户端（最方便）对 README.md 文件提交修改
4. 养成周期提交一次的习惯，以免赛博假死
